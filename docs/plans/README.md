# Plans Directory

This directory stores implementation plans and design plans that were written for specific development passes.

Status rules:

- Treat files here as historical or pending plans, not as automatically current instructions.
- Before executing any plan, reconcile project names, paths, commands, and assumptions against `docs/README.md`, `docs/STATUS.md`, and the current source tree.
- Do not update frozen playback-quality result artifacts from a plan unless the active goal explicitly asks for a baseline/report-set refresh.
- Prefer writing new long-lived decisions to `docs/DECISIONS.md` and new current-state summaries to `docs/STATUS.md`.
